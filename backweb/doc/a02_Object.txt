# 객체학습의 학습 순서
1. 클래스와 객체의 구분
2. 객체의 메모리 할당(stack/heap영역)
3. 클래스/객체의 구성요소에 대한 이해
	1) 필드
	2) 생성자 : 매개변수이해
		ps) 생성자/메서드 overloading 규칙
	3) 메서드
		매개변수
		리턴타입
		프로세스처리
	4) 필드 + 생성자 
	5) 필드 + 생성자 + 메서드
		생성자를 통해 필드값이 입력되고
		메서드1에 필드할당
		메서드2에 어떤 영향 미치고 활용.
		
4. 접근제어자에 대한 이해
	1) 종류 : public, protected, X(default), private
	2) 패키지 : 객체들을 묶는 묶음 단위
		- 같은 패키지 내 객체들은 X(default)이상일 때, 제한 없이 접근이 가능
		- private는 자체내 속성으로만 접근 가능/외부 객체에는 접근 불가
			보통 필드값에 설정된다.
		- 외부 패키지에서 객체가 public으로 선언되어 있으면 import한 후, 
			호출하여 사용가능하다.

5. 객체 vs 객체	
	1) 1:1객체
		클래스를 선언하고, 다른 클래스에서 호출하는 형식으로 처리하는 것으로
		객체로 만들어졌을 때, 객체 안에 객체로 처리되는 형태의 객체를 말한다.
		ex) 사용자 클래스가 소유하는 핸드폰 클래스 호출하여 처리 하는 등..
		
	2) 1:다객체
