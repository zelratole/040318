**Oracle 테이블 생성 및 CRUD 처리**

### 1. 흥미로운 도입

데이터베이스는 현대 비즈니스의 중심에 있습니다. 우리는 일상에서 온라인 쇼핑, 소셜 미디어, 은행 업무 등 
다양한 활동을 통해 끊임없이 데이터베이스와 상호작용하고 있습니다. 이 모든 것이 데이터베이스의 테이블에 저장되고
 관리되며, 이러한 테이블을 생성하고 데이터를 관리하는 것은 매우 중요한 기술입니다. 
 이번 학습에서는 Oracle 데이터베이스에서 테이블을 생성하고 CRUD(Create, Read, Update, Delete) 작업을 
 수행하는 방법을 배워보겠습니다.

### 2. 학습목표

이 학습을 통해 여러분은 다음과 같은 목표를 달성하게 됩니다:

- Oracle 데이터베이스에서 테이블을 생성하는 방법을 이해하고 실습한다.
- CRUD 작업을 수행하여 데이터를 효과적으로 관리하는 방법을 습득한다.
- SQL 명령어의 기초를 다지고, 이를 실무에 적용할 수 있는 능력을 기른다.

### 3. 주요 핵심 키워드와 핵심개념

- **테이블 생성 (CREATE TABLE)**: 테이블은 데이터베이스에서 데이터를 저장하는 기본 단위입니다. 
      각 테이블은 행(Row)과 열(Column)로 구성됩니다.
- **데이터 삽입 (INSERT INTO)**: 테이블에 새로운 데이터를 추가하는 작업입니다.
- **데이터 조회 (SELECT)**: 테이블에 저장된 데이터를 조회하는 작업입니다.
- **데이터 수정 (UPDATE)**: 테이블에 저장된 데이터를 수정하는 작업입니다.
- **데이터 삭제 (DELETE)**: 테이블에 저장된 데이터를 삭제하는 작업입니다.

### 4. 퀴즈문제/토론과제

**퀴즈문제:**
1. Oracle에서 테이블을 생성하는 SQL 명령어는 무엇인가요?
2. 데이터를 조회할 때 사용하는 SQL 명령어는 무엇인가요?
3. 특정 조건에 맞는 데이터를 삭제하기 위한 SQL 명령어는 무엇인가요?

**토론과제:**
- 데이터베이스 테이블의 설계 시 어떤 사항을 고려해야 할까요?
- CRUD 작업을 자동화할 수 있는 방법에는 어떤 것들이 있을까요?
- 대용량 데이터를 효과적으로 관리하기 위한 전략에는 무엇이 있을까요?

### 5. 실무적용사례

**사례 1: 전자상거래 시스템**
전자상거래 웹사이트에서는 고객, 제품, 주문 등의 데이터를 관리하기 위해 다양한 테이블을 사용합니다. 
고객 정보는 고객 테이블에, 제품 정보는 제품 테이블에, 주문 정보는 주문 테이블에 저장됩니다. 
이러한 테이블을 생성하고, 고객이 새로운 주문을 할 때마다 데이터를 삽입하고, 주문 상태를 업데이트하며, 
오래된 주문 기록을 삭제하는 등의 CRUD 작업이 일상적으로 이루어집니다.

**사례 2: 병원 관리 시스템**
병원 관리 시스템에서는 환자, 의사, 진료 기록 등의 데이터를 관리합니다. 환자 테이블에는 환자의 개인 정보가 저장되고,
 의사 테이블에는 의사의 정보가, 진료 기록 테이블에는 각 환자의 진료 기록이 저장됩니다. 
 새로운 환자가 등록될 때 데이터를 삽입하고, 환자의 진료 기록을 조회하며, 잘못 입력된 정보를 수정하고, 
 필요 없는 데이터를 삭제하는 작업이 필요합니다.
이러한 실무 사례들을 통해 Oracle 데이터베이스에서 테이블을 생성하고 CRUD 작업을 수행하는 
능력은 다양한 산업에서 필수적인 기술임을 알 수 있습니다. 이번 학습을 통해 이러한 능력을 길러보세요.
