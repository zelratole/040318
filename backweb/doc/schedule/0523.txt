### JSP JSTL, 상속, DAO

#### 1. 흥미로운 도입
웹 개발을 할 때 가장 흔히 사용되는 기술 중 하나는 JSP(JavaServer Pages)입니다. JSP는 동적인 웹 페이지를 
쉽게 작성할 수 있게 해주는 기술로, 서블릿과 함께 사용됩니다. 여기에 JSTL(JavaServer Pages Standard 
Tag Library)을 활용하면 더욱 간편하게 코드를 작성할 수 있습니다. 또한, 객체 지향 프로그래밍의 핵심 개념인 
상속과 데이터베이스 연동을 위한 DAO(Data Access Object) 패턴은 효율적인 코드 재사용과 데이터 처리에 
중요한 역할을 합니다. 이번 강의를 통해 JSP와 JSTL의 기본 사용법, 상속의 개념, 그리고 DAO 패턴을 학습하고 
실무에 어떻게 적용할 수 있는지 알아보겠습니다.

#### 2. 학습목표
- JSP와 JSTL을 사용하여 동적인 웹 페이지를 개발할 수 있다.
- 상속의 개념을 이해하고 이를 활용하여 객체 지향 프로그래밍의 효율성을 높일 수 있다.
- DAO 패턴을 이해하고 이를 사용하여 데이터베이스 연동을 효과적으로 수행할 수 있다.
- 실무 예제를 통해 학습한 내용을 실제 프로젝트에 적용할 수 있다.

#### 3. 주요 핵심 키워드와 핵심개념
- **JSP (JavaServer Pages)**: 서버 측에서 동적인 웹 페이지를 생성하는 기술.
- **JSTL (JavaServer Pages Standard Tag Library)**: JSP에서 사용할 수 있는 표준 태그 라이브러리로,
	 반복, 조건, 포맷 등의 작업을 쉽게 수행할 수 있음.
- **상속 (Inheritance)**: 객체 지향 프로그래밍에서 하나의 클래스가 다른 클래스의 속성과 메서드를 물려받는 
	기능.
- **DAO (Data Access Object)**: 데이터베이스와의 상호작용을 담당하는 객체로, 데이터 접근 로직과 비즈니스
	 로직을 분리하는 역할을 함.

#### 4. 퀴즈문제/토론과제
- **퀴즈문제**:
  1. JSP의 주요 기능은 무엇인가요?
  2. JSTL의 `<c:forEach>` 태그는 어떤 용도로 사용되나요?
  3. 상속의 주요 장점은 무엇인가요?
  4. DAO 패턴의 주요 목적은 무엇인가요?
  
- **토론과제**:
  1. JSP와 JSTL을 사용했을 때의 장점과 단점을 논의해보세요.
  2. 상속을 과도하게 사용했을 때 발생할 수 있는 문제점에 대해 토론해보세요.
  3. DAO 패턴을 사용하지 않고 직접 데이터베이스 연동을 처리하는 것과 DAO 패턴을 사용하는 것의 차이점에 대해 
  	논의해보세요.

#### 5. 실무적용사례
- **JSP와 JSTL을 사용한 웹 애플리케이션 개발**:
  한 전자상거래 웹 사이트에서 상품 목록을 동적으로 표시하기 위해 JSP와 JSTL을 사용했습니다. JSP 페이지에서는 
  상품 데이터를 JSTL의 `<c:forEach>` 태그를 사용하여 반복적으로 출력하였으며, 사용자에게 친숙한 UI를 
  제공했습니다.

- **상속을 활용한 코드 재사용**:
  한 프로젝트에서 여러 종류의 보고서를 생성해야 하는 요구사항이 있었습니다. 기본적인 보고서 생성 기능을 가진 
  `Report` 클래스를 만들고, 이를 상속받아 각기 다른 형식의 보고서를 생성하는 `SalesReport`, 
  `InventoryReport` 등의 클래스를 작성하여 코드의 재사용성을 높였습니다.

- **DAO 패턴을 적용한 데이터베이스 연동**:
  한 금융 애플리케이션에서는 데이터베이스와의 복잡한 상호작용을 쉽게 관리하기 위해 DAO 패턴을 도입했습니다. 
  `UserDAO`, `TransactionDAO` 등의 클래스를 만들어 데이터베이스 접근 로직을 캡슐화하고, 비즈니스 
  로직과의 분리를 통해 유지보수성을 높였습니다.이와 같은 예제를 통해 학습한 내용을 실무에 적용하여 효율적이고 
  유지보수가 용이한 코드를 작성할 수 있습니다.
