# 웹소켓을 통한 실시간 채팅
1. 웹환경에서 시용자간에 실시간 채팅을 여러 기능적인 필요로 할 때가 많다.
	실시간 채팅은 기본 웹 개발서버에서 추가적으로 chatting 모듈을 탑재하여
	처리되는 부분으로 스프링에서는 이에 대한 내용을 framework안에서 처리할 수 있다.
2. 주요 프로세스
	1) 로그인/일반사용자가 채팅화면을 클릭하고, 입장을 클릭, 화면에 입장된다.
	2) 사용자 인터페이스에 따라, 채팅방을 설정할 수도 있다.
	3) 채팅방에 입장하면 입장한 사용자로 하여금 메시지를 전송하거나
		전달 받을 수 있다.
	4) 종료 클릭시, 채팅방에서 나와 다른 채팅방에 가거나 다른 업무를 처리할 수 있다.
3. 개발 순서
	1) 컨테이너에 웹 chatting 소켓 설정
	2) java로 chatting handler 클래스 선언(back end)
		- 클라이언트에 의해서 접속/메시지전송/종료에 대한 내용을 처리해준다.
		- 상속한 클래스에서 제공하는 메서드에 의해서 여러가지 채팅기능을 처리한다.
		- 전역변수로 접속한 사용자의 socket session을 누적해서 저장처리한다.
		  접속 종료시 이, socket session에서 제외 처리된다.
		ex)
			    (소켓통신서버)
				
			기억    재현    (남원)			
		  
	3) 화면단(jsp)
		- chatting 화면 UI 구성(기본 구성)
			입장하는 사용자 입력, 입장버튼, 메시지입력/메시지 전송버튼, 메시지 내용 창, 퇴장 버튼 구현
		- 입장버튼 클릭시, js로 websocket 객체 생성 처리
		- socket서버에서 오는 push메시지로 메시지가 올 때마다 메시지 화면 처리
		- 메시지 입력 후, 전송 버튼 시, 해당 메시지를 서버로 전송
		- 종료버튼 클릭시, 서버와 연결을 종료 처리 된다..
