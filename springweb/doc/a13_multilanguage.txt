# 다국어 처리
1. 대부분의 global 기업은 각 업무 처리 프로그램이 현지 법인들과 같은
업무 처리 내용을 할 수 있도록 하고 있다. 초창기 웹 프로그래밍은 이와 같은
여러 나라에서 처리하는 업무에 맞게 해당 나라의 언어를 기반으로 다시 front단을
개발 하여 처리하는 경우가 많았다.
2. 이에 대하여 spring framework는 다국어 처리라는 기능을 통해서, 처리하는
언어는 코드값과 언어로 등록하여, 같은 웹페이지이지만 여러개의 언어 옵션에 따라
처리가 가능하게 만들었다. 이것을 다국어 처리라고 한다.
3. 다국어 처리 기본 프로세스
	1) 특정 나라마다 가지고 있는 웹브라우저의 언어 옵션에 따라 초기 화면에 해당
		옵션에 따른 언어가 출력되는 화면을 확인할 수 있다.
	2) 상단에 언어 선택 옵션에 따라 해당 언어로 화면 내용을 확인할 수 있다.
3. 개발 진행 순서
	1) 컨테이너에 언어 처리를 위한 설정 처리 - 언어 코드 파일 설정 위치 지정.
	2) 각 언어별 text 파일 작성
		key/value 형식의 code 파일을 만들어 처리한다.
		기본 설정 언어는 영어부터 처리한다.
		추가 언어 text 파일을 key 코드값을 기반으로 자동을 unicode를 생성한다.
	3) 화면단에 위 key 코드 기반으로 호출하게 처리한다.<spring:message>로 설정.
	4) 화면단에서 언어 변경 select 선택 처리 구현, 변경시 언어 치리 controller 호출
	5) 언어 처리 controller에서 선택한 언어에 따라 언어 설정 변경을 반영해서 해당 화면에서 변경되는
		언어로 확인할 수 있다.
