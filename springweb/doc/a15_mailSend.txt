# 메일발송
1. 웹서비스에서 메일 발송은 여러가지 경우에 활용된다. 기본적으로 등록된 회원에게 공지사항이나
	특정한 정보를 보내는 경우와 회사인사시스템에서 신규 사원에 대한 사원등록시 등록된 이메일로
	사원번호와 임시비번을 발송하는 경우, 회원관리시스템에서 id/pwd를 분실시 등록된 이메일로
	발송하는 등 다양한 경우에 활용된다.
2. 메일 발송 처리 프로세스
	1) 수신자/제목/내용 기타 정보를 입력한 후, 메일 발송을 클릭시
	2) 해당 서버에서 위 요청값을 받아서, 컨테이너에 등록된 메일 서버(SMTP)에 전달 처리
	3) 메일 발송 서버(SMTP-구글/다움/네이버 등등)에서는 해당 메일을 받아서 수신자에게 전송 처리
		자체 메일 발송 서버를 구축하여 사용할 수 있으니, 이 부분은 시스템 영역에 속함..
		- 도메인 등록/고정아이피/운영체제 메일 발송 서버 구축이 필요
	4) 메일 계정으로 해당 메일에 접속하여 내용 확인..
3. 개발 프로세스
	1) 메일 서버 접근(가입된 메일 계정으로) 하여, 보안적인 사항을 변경하고, 사용되는 계정과 비밀번호를 확인
	2) 컨테이너에 위 메일 발송 서버의 정보를 등록 처리
	3) controller/view로 기본 메일 발송 화면 구성
	4) 메일 발송시 service단에서 처리할 내용 구성
	5) 메일 발송 처리 확인..
